Features

(Almost) Unlimited and Secure Storage

Purifinity's core feature is scalability. The most important part to reach this goal is a storage design which can grow on demand. To meet this target Hadoop is used as basis for all storage.

Source files are stored as is in Hadoop for reference, but to store metrics and other information a more structured database is needed. For this purpose HBase is used together with Phoenix.

This kind of storage has two major advantages:

  • Linear scalability by adding more hardware with Hadoop and HBase as cluster.
  • Due to automated replication no backup strategy is needed, because parts of the cluster can fail without loosing data.

Graph Storage

For analysis and evaluation purpose Graph Storage is needed which is also scalable. All graph data is put into a DuctileDB which is also based on Hadoop/HBase and is scalable by design.

Powerful Server Architecture

Scalable storage is nice, but without the power to search and calculate meaningful information, it is useless. Based on Java Technologies and WildFly a clustered application server can be used to scale together with the storage to handle a lot of large projects over a long period of time.

Universal Web User Interface

Due to a big diversity of hardware where Purifinity may be needed to run on (Desktop PCs, Laptops,Tables, Smartphones...) a web based, reactive user interface is needed.

Purifinity's user interface is based on Angular and Bootstrap.

Plugin Architecture

Purifinity was meant to be language independent. To allow third parties to develop extensions, a plugin architecture is used.
